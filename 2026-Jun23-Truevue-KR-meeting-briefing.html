00:00 – 08:00 기술 교육
DCM 소개 및 기본 용어 정의
DCM(Digital Communication Module) = Prismax를 디지털화하는 모뎀 유사 칩. Prismax 후면 장착 시 EMR 연동, Remote View, 클라우드 Analytics 가능.

주요 용어: EMR/EHR (전자의무기록), PDMS (환자 데이터 관리 시스템, ICU 특화), Device Driver (통신 번역기 — Google Translate 비유), Middleware (Phillips 모니터링 등 중간 번역 레이어)
04:00 – 14:00 기술 교육
연결 아키텍처 — Serial vs HL7 vs Middleware
Serial/Binary 연결 (레거시): 1960년대 기술, 한국에서 현재 사용 중. Serial Driver 비용 기기당 최대 $20,000 + Prismax 소프트웨어 업데이트마다 재검증 필요.

Middleware: GE BOX 등 별도 장비 필요 (자본 비용 발생). EMR 공급사 또는 병원 IT 책임.

HL7 직접 연결 (차세대): DCM 장착 후 Ethernet/Wi-Fi로 EMR 직접 연동. Serial Driver, Middleware 불필요. Prismax 소프트웨어 업데이트 시 영향 최소. 병원 비용 절감 핵심 메시지.
05:00 – 10:00 ❗ 핵심 결정
영업팀 EMR 논의 범위 명확히 제한
영업팀/TS팀이 EMR 연동 상세 내용을 논의하면 안 됨 — 복잡성 오해, 판매 장애 유발 가능.

영업팀 허용 메시지: "HL7 무선 연동 및 미래보장형(future-proof) 연결 옵션 제공 가능"까지만.
EMR 상세 논의 시: Mitch(APAC) 또는 영진(한국 디지털 기술 리드)으로 즉시 에스컬레이션.
08:00 – 25:00 기술 교육
클라우드 아키텍처 — TrueVue Analytics & Remote View
클라우드 서버 위치: 프랑크푸르트 (독일), EU GDPR 준수 — TrueVue Analytics + Share Source(PD) 모두.

Remote View: 병원 네트워크 내 간호사 스테이션·스마트기기에서 Prismax 화면 실시간 모니터링 (인터넷 불필요).

클라우드 연결 조건: 병원 방화벽 규칙 면제(Firewall Rule Exemption) 필요 — 병원 IT 부서 승인 없이는 클라우드 불가. 현재 삼성서울·신촌세브란스 모두 미완료.

한국 법적 검토: EMR 연동/클라우드는 현재 법적 문제 없음. TVTM(Analytics)은 MFDS 명확화 후 추가 검토 필요 (병원별 정책 확인 요망).
25:00 – 36:00 전략 핵심
병원 IT 전략 — 전체 패키지 일괄 논의 vs 단계적 접근
논의 쟁점: Remote View만 먼저 활성화 → 단계적으로 Analytics(방화벽 개방) 추진 vs 처음부터 전체 패키지 요청.

최종 방향 (Mitch/West): 병원 IT와 단 한 번의 종합 논의로 모든 요구사항 전달. Remote View만 먼저 활성화하면 이후 IT에 방화벽 추가 개방 요청이 극히 어려워짐. 임상부서는 Remote View로 만족하므로 IT 재접촉 기회 상실.

제안 체크리스트: ① 기기 네트워크 설치 ② 방화벽 규칙 면제 ③ MAC주소/IP 화이트리스트 등록 ④ Wi-Fi 설정 ⑤ 클라우드 연결 테스트 — 5개 항목 완료 시 점검 완료.

Internal Risks

내부 리스크 및 주의사항

🚨 한국 병원 방화벽 정책이 매우 엄격

Jinsik 현장 경험 포함, 이전 회사에서도 동일 이슈. 대부분 대형 병원이 강한 인터넷 차단 정책 보유. 초기 설득에 시간이 걸릴 수 있음. 방화벽 전략 오프라인 논의(Mitch·West·Jinsik) 선행 필수.

🚨 신촌세브란스 담당 의사가 리포트를 오래 기다리고 있음

비즈니스 임팩트 리스크. 보고서는 준비됐지만 메시지 정렬 문제로 미전달 상태. Rokyoung 주관 소규모 정렬 미팅 즉시 진행 후 전달 필요. 더 이상 지연 시 고객 신뢰 손상.

⚠️ 리포트 인사이트 메시지 현지화 필요

한국 CRRT 관행(7일 필터 세트 등)이 글로벌 기준과 달라, 글로벌 리포트 기준 그대로 전달 시 고객에게 부정적 메시지로 해석될 가능성. KSN CRRT 가이드라인 맥락에서 현지화된 해석 제공 필수.

⚠️ 메시지 일관성 — "누가 무엇을 말하는가" 경계 명확히

영업팀이 기술 영역(EMR·방화벽)을 침범하거나, 기술팀이 영업 메시지와 다른 내용을 고객에게 전달하면 복잡해짐. 영업팀은 High-level 메시지만, 기술 논의는 반드시 Youngjin / Mitch가 담당. 역할 경계 훈련 전 사전 정렬 필요.

핵심 메시지 — "Future-Proof Connectivity"

한국 영업팀이 고객에게 전달할 핵심은 단 하나: Vantive Connect Critical Care는 현재와 미래를 모두 대비하는 연결 옵션을 제공합니다.

HL7 무선 연동 → 병원 비용 절감 (Serial Driver $20K/기기 불필요) · KSN CRRT 가이드라인 달성 도구 · Analytics로 치료 성과 지속 개선 · 경쟁사 대비 차별화 기반 구축.

깊은 기술 논의는 영진 + Mitch가 함께합니다. 영업팀은 고객 니즈를 먼저 발굴하세요.
